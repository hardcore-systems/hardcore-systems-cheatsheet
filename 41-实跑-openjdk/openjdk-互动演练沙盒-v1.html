Java HotSpot VM 核心运行机制仿真实验室

通过两个独立的高仿真实验模块，深入演练 HotSpot VM 底层逻辑：第一关为 JMM 内存模型指令重排与硬件屏障实验室，展示指令重排与 Store Buffer 延迟刷新导致的并发可见性破坏，以及插入内存屏障 (StoreLoad) 后强制冲刷缓存解决异常的过程；第二关为 ZGC 彩色指针与读屏障自愈实验室，模拟垃圾回收阶段性并发迁移（Relocate）、转发表（Forwarding Table）映射以及应用线程触发读屏障进行指针自愈（Self-healing）转为 Remapped 状态的物理逻辑。

并发异常检测触发！
发生了 r1 == 0 && r2 == 0 异常！这是典型的 JMM 规范违背现象。硬件成因为 CPU 对读写指令进行了流水线乱序重排，或者写入操作滞留在处理器私有的 Store Buffer 中，尚未刷新到全局共享的高速缓存中，导致对端线程不可见。
