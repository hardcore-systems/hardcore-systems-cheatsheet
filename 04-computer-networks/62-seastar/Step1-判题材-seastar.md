# Step1-判题材-seastar: Shard-per-Core 极速无锁高性能网络框架

## 1. 题材重要性与壁垒分析
Seastar 是一款超高性能的异步 C++ 应用程序框架，由 ScyllaDB 团队开发。它首创了 Shard-per-core 架构，为现代海量吞吐、极低延迟的服务器程序树立了技术标杆。其通过彻底的 CPU 绑定、无共享（Shared-Nothing）模型、内存本地化和异步非阻塞 Promise/Future 链，将服务器吞吐榨干至物理硬件极限，完美规避了多核架构下的锁争抢和多线程上下文切换瓶颈。

## 2. 知识模块组织 (M1-M6) & 配色映射
- **M1: Shard-per-core 极速无锁调度架构 (M1: #4B5F7A)** - CPU 独占与线程绑定、无锁内存分配器、协作式协程调度、跨核无锁环形队列。
- **M2: 异步非阻塞编程模型 (M2: #6B8272)** - Future/Promise 核心、链式 Continuation 优化、C++20 协程集成、微秒级任务调度队列。
- **M3: 高性能网络与磁盘 I/O 引擎 (M3: #9C6666)** - io_uring 引擎、零拷贝 Packet 缓存、DMA 物理 Direct I/O、磁盘 I/O 公平队列、DPDK 用户态协议栈。
- **M4: 无锁并发与多核拓扑 (M4: #7A7A7A)** - 原子无锁共享结构、RCU 读写分离、多核并发屏障、缓存行对齐对冲伪共享、NUMA 感知内存。
- **M5: 分布式应用支撑与微服务 (M5: #9A825A)** - 跨 Shard 分布式服务、连接限流管理器、背压流控自愈、零拷贝极速 HTTP/JSON 解析。
- **M6: 系统运行参数调优与诊断 (M6: #755B77)** - CPU 挂起检测与慢任务监控、异步堆栈回溯调试、内核指标遥测、C++ 编译期优化参数字典。

## 3. J-Ladder 阶梯设计逻辑
- **L0: 底层物理绑定** (M1-M4) - CPU 亲和性硬绑定，物理核心独占，NUMA 本地化内存，SPSC 跨核环形队列。
- **L1: 异步调度引擎** (M2-M3) - Promise/Future 调度器，io_uring/DPDK 零拷贝网卡驱动，C++20 协程转换。
- **L2: 微服务与诊断** (M5-M6) - 跨 Shard 状态分发，背压自愈，Reactor 挂起遥测监控，异步堆栈诊断。
