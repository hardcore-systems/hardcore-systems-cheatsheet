# Step1-判题材-duckdb: 嵌入式分析型 SQL 引擎

## 1. 题材重要性与壁垒分析
DuckDB 是一款嵌入式、进程内的列式分析型（OLAP）SQL 数据库引擎，被称为“OLAP 领域的 SQLite”。它开创性地将高性能向量化计算引擎整合进嵌入式轻量级库中，解决了大厂海量数据单机即席分析、数据科学集成（如 Pandas、Polars 无缝零拷贝对接）以及无服务器架构下极速分析的壁垒。

## 2. 知识模块组织 (M1-M6) & 配色映射
- **M1: 列式存储与内存物理布局 (M1: #4B5F7A)** - 行组划分（Row Groups）、数据块物理分配、列式段存储管理、Zone Map 区间过滤。
- **M2: 向量化计算引擎核心 (M2: #6B8272)** - 向量化计算核心、行式火山模型对比、CPU SIMD 寄存器硬件加速、类型动态投影转换。
- **M3: 事务管理与 Epoch MVCC (M3: #9C6666)** - Epoch 事务管理器、可见性判别读路径、逻辑删除与版本链、列式数据归并压实与压缩。
- **M4: 并行计算与查询管线 (M4: #7A7A7A)** - Push 模式执行管线、并行扫描与工作窃取、内存哈希连接算法、外排内存溢出落盘机制。
- **M5: 数据格式与物理接口 (M5: #9A825A)** - Parquet 零拷贝读取、Arrow 内存格式无缝映射、CSV/JSON 极速并行解析。
- **M6: 系统扩展与性能遥测 (M6: #755B77)** - 目录 Catalog 树、UDF 扩展 SDK、执行计划 Profiling 遥测、资源配额硬上限限制。

## 3. J-Ladder 阶梯设计逻辑
- **L0: 物理数据底座** (M1-M3) - 列式 Block 段布局，基于 Epoch 的 MVCC 可见性判定，以及轻量 WAL。
- **L1: 向量计算与执行** (M2-M4) - Vector 向量化（1024 元素）算子执行，Push模型执行流水线，SIMD 编译加速。
- **L2: 外设接口与诊断** (M5-M6) - Arrow/Parquet 零拷贝互通，插件与 UDF 扩展，算子级执行 Profiling 指标遥测。
