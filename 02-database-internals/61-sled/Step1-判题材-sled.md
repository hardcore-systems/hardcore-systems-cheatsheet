# Step1-判题材-sled: Rust 现代嵌入式 B-Tree 键值存储引擎

## 1. 题材重要性与壁垒分析
Sled 是一款现代的、采用 Rust 开发的 Flash-Native（闪存友好型）嵌入式键值数据库引擎。它独特的架构融合了无锁 B-Link 树（B-Link Tree）与日志结构（Log-Structured）页存储。通过无锁并发读写与 Delta 链表式逻辑更新，彻底平抑了闪存介质上的写放大与并发锁争抢，是高并发、嵌入式强一致存储方向的技术巅峰。

## 2. 知识模块组织 (M1-M6) & 配色映射
- **M1: 无锁 B-Link Tree 与逻辑结构 (M1: #4B5F7A)** - 无锁 B-Link 树核心、页表原子 CAS 更新、叶子/内部节点物理结构、读写并发折衷。
- **M2: 日志结构页存储与 Delta 链 (M2: #6B8272)** - 日志物理追加写、页 Delta 链表设计、段映射重定向、页合并（Consolidation）压实、WAL 刷盘。
- **M3: 并方页分裂时序与恢复 (M3: #9C6666)** - 叶子页溢出物理分裂、分裂期右链接无锁遍历、异步父节点路由更新、冲突回滚机制、冷启动日志重建。
- **M4: LSN 机制与 Epoch 垃圾回收 (M4: #7A7A7A)** - 逻辑 Log Sequence Number、Epoch 内存回收（EBR）、段 GC 清理与复制搬迁、写放大生命周期调优。
- **M5: 嵌入式事务与并发 API (M5: #9A825A)** - 交互式事务 API、键范围冲突检测、会话内读己所写缓冲、多线程安全句柄。
- **M6: 系统运行参数调优与诊断 (M6: #755B77)** - 缓存驱逐管理、Failpoint 异常注入测试、运行时监控遥测、高吞吐读写参数调优字典。

## 3. J-Ladder 阶梯设计逻辑
- **L0: 页存储与日志层** (M1-M2) - Page Table 原子寻址，内存 Delta 链，LSN 刷盘与 WAL 追加。
- **L1: 树形拓扑与并发** (M3-M4) - B-Link 树无锁右链接，页分裂，Epoch（EBR）物理垃圾回收。
- **L2: 事务与接口调优** (M5-M6) - 乐观交互事务 API，Segment GC 重建，运行时 Failpoint 测试与监控遥测。
