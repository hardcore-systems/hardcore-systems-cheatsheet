# Step1-判题材-dapr: 云原生 Sidecar 分布式应用运行时

## 1. 题材重要性与壁垒分析
Dapr (Distributed Application Runtime) 是微软开源的 CNCF 孵化级云原生分布式应用运行时。它通过 Sidecar（边车）架构将分布式系统的核心共性能力（状态管理、发布订阅、服务调用、绑定、Actor 等）抽象为统一的 gRPC/HTTP API，实现了应用逻辑与底层基础设施的物理解耦。其核心技术壁垒在于：极简的协议抽象设计（多语言通用）、高度可插拔的组件生态组件字典（屏蔽底层 Redis/Kafka/Postgres 差异）、以及集群级多 Sidecar 协同下的 Actor 分布式状态机与位置路由引擎。

## 2. 知识模块组织 (M1-M6) & 配色映射
- **M1: Sidecar 架构与 API 拦截 (M1: #4B5F7A)** - App-to-Sidecar 本地 gRPC/HTTP 通信、端口生命周期拦截、流量边车双向代理、API 统一抽象协议、应用无感集成机制。
- **M2: 可插拔状态管理与分布式锁 (M2: #6B8272)** - 状态存储组件设计、多后端一致性对齐、ETag 并发冲突乐观锁、分布式锁 API、批量与事务状态操作。
- **M3: 发布订阅与事件驱动 (M3: #9C6666)** - CloudEvents 规范协议包装、可插拔消息代理中间件（Redis/Kafka）、订阅端点生命周期注册、重试回退机制、死信队列（DLQ）流转。
- **M4: Actor 模型与分布式状态机 (M4: #7A7A7A)** - 虚拟 Actor 激活与休眠生命周期、Actor 状态一致性保证、Placement 服务多实例调度定位、Actor 定时器与提醒器、并发锁与队列化执行。
- **M5: 声明式绑定与外部集成 (M5: #9A825A)** - 输入绑定事件源监听、输出绑定外部资源触发、声明式元数据组件配置、双向数据流协议管道、中间件管道过滤网关。
- **M6: 系统安全与弹性可观测 (M6: #755B77)** - 双向 TLS (mTLS) 身份认证、SPIFFE 统一身份管理体系、弹性重试与熔断机制、W3C 分布式追踪链路传播、OpenTelemetry 指标采集。

## 3. J-Ladder 阶梯设计逻辑
- **L0: 边车通信与状态解耦** (M1-M2) - Dapr 边车本地 API 桥接，状态一致性乐观锁，事务状态存储。
- **L1: 异步事件与分布式计算** (M3-M4) - CloudEvents 消息投递，可插拔 Pub/Sub 代理，分布式 Actor 位置定位与 Placement 生命周期。
- **L2: 桥接集成与弹性安全** (M5-M6) - 双向声明式绑定桥接，mTLS 身份鉴权，熔断降级与 OTel 分布式追踪遥测。
